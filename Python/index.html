Base Types

Base types are the fundamental data types in Python, representing basic values that can be used to construct more complex structures. Understanding these types is essential for efficient programming.

  • Integers: Integers are whole numbers that can be positive, negative, or zero. They are used for counting and indexing, making them a vital type for loop counters and array indexing.
  • Floats: Floats are numbers that contain decimal points or are expressed in exponential form. They are crucial for mathematical calculations requiring precision, such as scientific computations or financial calculations.
  • Booleans: Booleans represent truth values and can either be True or False. They are widely used in conditional statements and logical operations, enabling decision-making in code.
  • Strings: Strings are sequences of characters used to represent textual data. They can be manipulated in various ways, such as concatenation, slicing, and formatting, making them versatile for handling user input and output. 

# Integer
x = 42

# Float
y = 3.14

# Boolean
is_true = True

# String
message = "Hello, World!"

Container Types

Container types in Python are designed to hold collections of items, allowing for efficient data management. Each type has its characteristics, making them suitable for different use cases.

  • Lists: Lists are ordered collections that can hold a mix of different data types. They are mutable, meaning their contents can be changed after creation. Lists are ideal for maintaining sequences of items, such as user inputs or results of computations.
  • Tuples: Tuples are similar to lists but are immutable, meaning once they are created, their contents cannot be altered. This property makes tuples useful for fixed collections of items, such as coordinates or settings that should not change.
  • Dictionaries: Dictionaries store key-value pairs in an unordered manner, where each key must be unique. They are extremely useful for associative arrays and allow for quick lookups of values based on keys, making them ideal for representing structured data like user profiles or configuration settings.
  • Sets: Sets are unordered collections of unique elements. They are primarily used to eliminate duplicate values and perform mathematical set operations such as union, intersection, and difference. Sets are particularly useful when the uniqueness of elements is essential.

# List
fruits = ["apple", "banana", "cherry"]

# Tuple
coordinates = (10, 20)

# Dictionary
person = {"name": "John", "age": 30}

# Set
unique_items = {1, 2, 3}

Identifiers

Identifiers in Python are names assigned to variables, functions, classes, and other entities. Proper naming conventions enhance code readability and maintainability.

  • Identifiers must start with a letter (either lowercase or uppercase) or an underscore (_). This rule ensures that identifiers are easily distinguishable from numerical values.
  • After the first character, identifiers can include letters, digits (0-9), and underscores. However, they cannot start with a digit, as this could cause confusion in interpretation.
  • Identifiers are case-sensitive, meaning myVariable and myvariable are treated as two distinct identifiers. This feature allows for greater flexibility but also requires careful naming to avoid confusion.
  • Avoid using Python's reserved keywords, such as if, else, while, etc., as identifiers to prevent syntax errors.

# Naming rules:
# - Must start with a letter (a-z, A-Z) or underscore (_)
# - Followed by letters, digits (0-9), or underscores
# - Case-sensitive
my_variable = 10

Variable Assignments

Variable assignments in Python are how you store data for later use. The equals sign (=) is used to assign values to variables. Understanding variable assignments is crucial for effective programming, as it helps manage data flow within your applications.

  • Single assignment: This occurs when you assign one value to a variable. It is the most common form of assignment.
  • Multiple assignment: Python allows you to assign values to multiple variables simultaneously, making your code concise and enhancing readability. This feature is particularly useful for initialising several related variables in one line.

# Single assignment
x = 10

# Multiple assignment
a, b, c = 1, 2, 3

Conversions

Type conversion in Python allows you to change the data type of a value. This is especially useful when performing operations that involve multiple data types. Python provides built-in functions to facilitate these conversions, making it easy to work with various data types interchangeably.

  • String conversion: The str() function converts a value into a string format. This conversion is useful when you want to concatenate numbers and text or display them together.
  • Integer conversion: The int() function converts a string or a float to an integer, truncating any decimal values. This is useful for situations where whole numbers are required.
  • Float conversion: The float() function converts a string or an integer into a floating-point number, which is beneficial when performing mathematical operations that require decimal precision.

# Converting to string
num = 42
str_num = str(num)

# Converting to integer
str_value = "10"
int_value = int(str_value)

# Converting to float
float_value = float(str_value)

Boolean Logic

Boolean logic is fundamental in programming for controlling the flow of execution based on conditions. Python provides several logical operators to combine or modify boolean values, enabling complex decision-making in your code.

  • AND: The and operator returns True if both operands are true; otherwise, it returns False. This is useful for ensuring that multiple conditions must be met.
  • OR: The or operator returns True if at least one of the operands is true; it returns False only if both operands are false. This operator is helpful when you want to check for at least one condition being true.
  • NOT: The not operator negates the boolean value of its operand. If the operand is true, not makes it false, and vice versa. This is useful for reversing the logic of conditions.

# Logical operators
x = True
y = False

# AND
result = x and y  # False

# OR
result = x or y  # True

# NOT
result = not x  # False

Statements Blocks

In Python, statements are grouped into blocks, which are defined by indentation. Properly structuring your code into blocks makes it more readable and maintains the logical flow.

  • Code within a block must be indented consistently to indicate that it belongs to the same statement. This feature of Python enforces clean and readable code formatting.
  • Blocks are commonly used in conditional statements, loops, and function definitions. Understanding how to create and use blocks is crucial for effective Python programming.

# Defining a block
if x > 10:
    print("x is greater than 10")
    print("This is inside the block")
print("This is outside the block")

Modules/Names Imports

Python modules are files containing Python code, which can include functions, classes, and variables. Using modules allows for code reusability and organisation.

  • Importing an entire module: The import statement allows you to include an entire module in your code, enabling access to all its functions and variables.
  • Importing specific functions: You can import specific functions from a module using the from ... import ... syntax. This approach can make your code cleaner by reducing the need to prefix function calls with the module name.
  • Once imported, you can use the functions or variables defined in the module as if they were part of your code. This feature promotes modular programming, which is essential for maintaining and scaling larger applications.

# Importing an entire module
import math

# Importing specific functions
from math import sqrt, pi

# Using an imported function
result = sqrt(16)  # 4.0

Conditional Statements

Conditional statements allow you to execute certain blocks of code based on specific conditions. They are essential for controlling the flow of your program and enabling dynamic behaviour.

  • The if statement checks a condition and executes the corresponding block if the condition is true.
  • The elif statement (short for "else if") allows you to check multiple conditions sequentially, executing the block of the first true condition.
  • The else statement executes a block of code if none of the preceding conditions were true, providing a default action.
  • Understanding how to structure conditional statements effectively enables you to implement complex decision-making processes in your code.

if x > 10:
    print("x is greater than 10")
elif x == 10:
    print("x is equal to 10")
else:
    print("x is less than 10")

Maths

Mathematics in Python encompasses basic arithmetic operations and more complex mathematical functions. Understanding these operations is foundational for performing calculations and developing algorithms.

  • Arithmetic operations: Python supports standard operations like addition (+), subtraction (-), multiplication (*), and division (/). These operations can be applied to integers and floats, making it easy to perform calculations.
  • Using math functions: The math module provides access to advanced mathematical functions, such as sqrt() for square roots, sin() for sine, and others. Importing the math module expands your capabilities significantly, allowing for complex mathematical computations.
  • Mastering these operations and functions is crucial for scientific programming, data analysis, and any application requiring numerical computations.

# Arithmetic operations
a = 5
b = 3

sum = a + b          # Addition
diff = a - b         # Subtraction
product = a * b      # Multiplication
quotient = a / b     # Division

# Using math functions
import math
sqrt_value = math.sqrt(25)  # Square root

Loops

Loops in Python are structures that allow you to repeat a block of code multiple times, facilitating automation and reducing redundancy in your code.

  • For loops: The for loop iterates over a sequence (like a list or a string) or generates a sequence using the range() function. This loop is ideal for scenarios where you know the number of iterations in advance.
  • While loops: The while loop continues to execute as long as a specified condition is true. This loop is particularly useful when the number of iterations is not predetermined and depends on dynamic conditions.
  • Both types of loops can control the flow of your program effectively, enabling you to automate repetitive tasks and manage collections of data efficiently.

# For loop
for i in range(5):
    print(i)

# While loop
i = 0
while i < 5:
    print(i)
    i += 1

Functions

Functions in Python are reusable blocks of code that perform specific tasks. They allow for better organisation of code, enabling modular programming.

  • Functions are defined using the def keyword, followed by the function name and parentheses that may contain parameters.
  • Parameters allow functions to accept inputs, making them versatile and applicable in various situations. You can also define default values for parameters, providing flexibility in how the function can be called.
  • Once defined, a function can be called from anywhere in your code, reducing redundancy and promoting code reuse. This capability is crucial for maintaining and scaling larger applications.

# Function definition
def greet(name):
    return f"Hello, {name}!"

# Calling the function
message = greet("Alice")  # "Hello, Alice!"

String Operations

String operations in Python involve manipulating textual data. Strings are widely used for user input, output, and data processing, making string operations fundamental to programming.

  • String concatenation: You can combine two or more strings using the + operator. This operation is essential for constructing messages and formatting output dynamically.
  • String slicing: Slicing allows you to extract a portion of a string using a specific range of indices. This feature is useful for manipulating substrings, such as extracting names or specific data points from a larger text.
  • String formatting: Python supports various methods for formatting strings, such as f-strings, which allow you to embed expressions inside string literals. This capability enhances the readability and maintainability of your output.

# String concatenation
greeting = "Hello, " + "World!"

# String slicing
substring = greeting[0:5]  # "Hello"

# String formatting
name = "Alice"
formatted = f"Hello, {name}!"  # f-string

File Handling

File handling in Python involves reading from and writing to files, enabling persistent data storage. Mastering file operations is essential for applications that require data logging, configuration management, or user data processing.

  • Files can be opened using the open() function, which requires the filename and the mode (e.g., read r, write w). The with statement is recommended for file operations, as it ensures that the file is properly closed after its suite finishes.
  • When writing to a file, you can use the write() method to add text. This process allows for saving data, logging messages, or generating output files.
  • To read from a file, the read() method retrieves the file’s contents, which can then be processed or displayed. Proper error handling should be implemented to manage scenarios where the file may not exist.

# Writing to a file
with open("file.txt", "w") as file:
    file.write("Hello, World!")

# Reading from a file
with open("file.txt", "r") as file:
    content = file.read()

Errors and Exceptions

Error handling in Python is vital for building robust applications. It allows you to manage unexpected issues gracefully, preventing your program from crashing.

  • Exceptions are errors that occur during the execution of your code. Python provides a mechanism to handle these exceptions using the try and except blocks.
  • The try block contains code that may potentially raise an exception. If an exception occurs, Python jumps to the except block, where you can define how to handle the error.
  • This structured approach to error handling enhances user experience by providing informative messages or alternative flows rather than abrupt crashes.

try:
    result = 10 / 0
except ZeroDivisionError:
    print("You can't divide by zero!")

List Comprehensions

List comprehensions provide a concise way to create lists in Python. They enhance readability and can reduce the amount of code you write for generating lists.

  • A list comprehension consists of an expression followed by a for clause and optionally, additional if clauses. This structure allows you to create lists in a single line.
  • List comprehensions can improve performance by avoiding the need for explicit loops, making your code faster and more efficient.
  • They are especially useful for transforming data or filtering elements based on conditions, making your data processing tasks simpler and cleaner.

# List comprehension
squares = [x**2 for x in range(10)]  # [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]

Lambda Functions

Lambda functions are small, anonymous functions defined using the lambda keyword. They are useful for creating simple functions on-the-fly without formally defining them using def.

  • Lambda functions can take any number of arguments but can only have a single expression. This feature makes them ideal for short operations or callbacks.
  • They are often used in higher-order functions like map(), filter(), and sorted(), where you need to pass a function as an argument.
  • While lambda functions are convenient, they should be used judiciously, as overusing them can lead to less readable code.

# Lambda function
add = lambda x, y: x + y
result = add(5, 3)  # 8

Dictionaries and Sets

Dictionaries and sets are essential data structures in Python, each serving distinct purposes for data storage and management.

  • Dictionaries: Dictionaries consist of key-value pairs, enabling fast lookups, insertions, and deletions. Keys must be unique, and they can be of any immutable data type (like strings, numbers, or tuples). This structure is ideal for representing structured data, such as user profiles or configuration settings.
  • Sets: Sets are collections of unique elements, allowing you to store items without duplicates. They support mathematical operations like union, intersection, and difference, making them useful for tasks involving unique items or membership tests.
  • Both data structures enhance the capabilities of your applications by providing efficient ways to manage and manipulate data.

# Dictionary
person = {"name": "Alice", "age": 25}
print(person["name"])  # "Alice"

# Set
unique_numbers = {1, 2, 3, 3}
print(unique_numbers)  # {1, 2, 3}

Object-Oriented Programming (OOP)

Python supports Object-Oriented Programming (OOP), which organises code around objects rather than actions, and data rather than logic. It promotes code reusability and is ideal for modeling real-world systems. Key OOP concepts include:

  • Classes: Blueprints for creating objects (instances). A class defines the properties and behaviours (methods) of an object.
  • Inheritance: A mechanism for one class (child class) to inherit the properties and methods of another class (parent class), allowing code reuse and a hierarchical organisation.
  • Polymorphism: The ability of different classes to be treated as instances of the same class through shared interfaces, especially useful when classes override inherited methods.

# Class definition
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return f"{self.name} makes a sound"

# Inheritance
class Dog(Animal):
    def speak(self):
        return f"{self.name} barks"

Functional Programming

Python supports functional programming, a paradigm where programs are constructed by applying and composing functions. This style emphasises the use of pure functions, immutability, and avoiding state changes. Common functional programming tools in Python include:

  • Lambdas: Anonymous, inline functions defined using the lambda keyword, often used for short operations.
  • Map: Applies a function to each element of a sequence and returns a list of results.
  • Filter: Filters a sequence based on a condition, returning a new list of elements that meet the condition.
  • Reduce: Applies a function cumulatively to the items of a sequence to reduce it to a single value (requires the functools module).

# Using lambda and map
numbers = [1, 2, 3, 4]
squares = list(map(lambda x: x ** 2, numbers))  # [1, 4, 9, 16]

# Using filter
evens = list(filter(lambda x: x % 2 == 0, numbers))  # [2, 4]

Decorators

Decorators are a powerful feature in Python that allow you to modify the behaviour of functions or methods. They are higher-order functions that take another function and extend or alter its behaviour without explicitly modifying it. Common use cases include logging, timing, and access control. 

# Example of a decorator
def decorator(func):
    def wrapper():
        print("Before the function call")
        func()
        print("After the function call")
    return wrapper

@decorator
def say_hello():
    print("Hello!")

say_hello()

# Output:
# Before the function call
# Hello!
# After the function call

Exceptions

Exceptions in Python are used to handle errors that may occur during program execution. Python provides a range of built-in exception types for different error conditions. Exception handling helps to ensure that a program can gracefully deal with unexpected situations without crashing:

  • Try-except: Code that might raise an error is placed inside the try block. If an exception occurs, it is caught by the corresponding except block.
  • Finally: A finally block can be used to specify cleanup actions that should run whether or not an exception occurred.

try:
    # Code that might raise an exception
    result = 10 / 0
except ZeroDivisionError:
    print("Cannot divide by zero")
finally:
    print("This runs no matter what")

Generators

Generators are a special type of function in Python that yield a sequence of values one at a time, rather than returning them all at once. This makes generators highly memory efficient for working with large datasets or infinite sequences. They use the yield keyword instead of return. 

# Generator example
def count_up_to(max_value):
    count = 1
    while count <= max_value:
        yield count
        count += 1

# Using the generator
for number in count_up_to(5):
    print(number)
